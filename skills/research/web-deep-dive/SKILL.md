# SKILL: web-deep-dive

## Trigger
사용자가 특정 주제의 심층 리서치를 요청하고, 여러 출처를 종합해야 할 때.
("~에 대해 자세히 조사해줘", "최신 동향 정리해줘" 등)

## Procedure
1. 질문을 2~4개의 하위 질문으로 분해한다.
2. 각 하위 질문을 병렬로 웹 검색하고, 1차 출처 위주로 수집한다.
3. 주장마다 출처를 붙여 교차 검증한다 (단일 출처 주장은 "미확인"으로 표기).
4. `knowledge/<주제>-<날짜>.md`에 출처 링크와 함께 저장한다.
5. 요약 → 상세 → 출처 순으로 사용자에게 보고한다.

## Pitfalls
- 단일 출처를 사실로 단정하기 → 최소 2개 교차 확인.
- 오래된 정보 → 날짜를 항상 확인하고 명시.

# SKILL: debug-loop

## Trigger
버그·에러·예상과 다른 동작을 만났을 때 (수정 제안 전에 발동).

## Procedure
1. 재현 조건을 먼저 확정한다 (재현되는 최소 케이스).
2. 가설을 세우고, 로그/이분 탐색으로 원인을 좁힌다.
3. 근본 원인을 찾은 뒤에만 수정한다 (증상 땜질 금지).
4. 재현 케이스가 통과하는지 확인한다.
5. 원인→해결 경로를 Pitfalls와 MEMORY.md에 기록한다.

## Pitfalls
- 원인 파악 전 수정부터 → 항상 재현→가설→검증 순.
- 한 번에 여러 곳 수정 → 변경을 최소·격리.

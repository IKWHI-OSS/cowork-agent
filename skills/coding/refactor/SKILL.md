# SKILL: refactor

## Trigger
코드 정리·구조 개선 요청 (동작 변경 없이).

## Procedure
1. 리팩터링 전 테스트가 통과하는지 확인한다 (없으면 특성화 테스트 추가).
2. 한 번에 하나의 변경만 적용한다.
3. 매 단계 후 테스트를 재실행한다.
4. 기존 스타일·컨벤션을 따른다.
5. 동작이 동일함을 확인하고 보고한다.

## Pitfalls
- 테스트 없이 대규모 변경 → 먼저 안전망 확보.
- 리팩터링에 기능 변경 섞기 → 분리.

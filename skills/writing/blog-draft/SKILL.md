# SKILL: blog-draft

## Trigger
블로그/아티클 초안 작성 요청.

## Procedure
1. 주제·타깃·목표 길이를 확인한다.
2. 개요(헤딩 구조)를 먼저 잡고 사용자 확인을 받는다.
3. 섹션별로 초안을 작성한다 (도입-본문-마무리).
4. 제목·메타 설명 후보를 제안한다.
5. `knowledge/` 또는 지정 폴더에 저장.

## Pitfalls
- 개요 없이 바로 작성 → 구조가 흔들림. 개요 먼저.
- 일반론 나열 → 구체 예시/데이터로 보강.
